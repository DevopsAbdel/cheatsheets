# Courses Vault — 2026 Edition

## Core Philosophy

A good Courses Vault should be:

- Searchable
- Future-proof
- Minimal
- Action-oriented
- Easy to archive
- AI-friendly

Avoid:
- Random downloads folders
- Course names with emojis/special chars
- Huge nested folders
- “Final_v2_REAL_FINAL”

# Best Learning Workflow

## 1. Capture
Take lightweight notes during learning.

Capture:
- Key ideas
- Frameworks
- Mental models
- Commands
- Mistakes

## 2. Compress
After each module create:

```txt
5_bullet_summary.md
```

Ask:
- What matters most?
- What is actionable?
- What would I teach someone?

## 3. Apply
Create mini-projects immediately.

```txt
Projects/
├── AI_Chatbot/
├── RAG_Search_App/
├── Landing_Page/
└── Automation_Workflow/
```

## 4. Retrieve
Use:
- Flashcards
- Quizzes
- Active recall
- Teaching

## 5. Systemize
Turn repeated concepts into:
- Templates
- Cheat sheets
- SOPs
- Reusable workflows

# Best File Naming Rules

Use:
```txt
lowercase_or_PascalCase
```

Avoid:
- spaces everywhere
- symbols
- accents
- long names

Good:
```txt
ai_agents_architecture.md
PromptEngineering_Basics.pdf
```

# Best File Types

| Purpose | Best Format |
|---|---|

| Data | .csv |
| Structured docs | .docx |
| Final exports | .pdf |
| Code | .py / .js |
| Templates | .md |
| Flashcards | .csv / Anki |

Markdown (.md) is the best long-term format.

Do NOT organize by:
- Platform
- Random year
- Download date

- Skill
- Outcome
- Career path
- Domain

That scales much better long term.
